---
title: "Search"
api: "POST /feedback/search"
---

<ParamField body="project_name" type="string" required>
  name of Sieve project
</ParamField>

This endpoint lets you retrieve all stored feedback you've submitted that matches a certain query. 
The query is specified in the body of the request based on the [feedback structure](/reference/api/feedback/intro). 
The `data` field is influenced by the feedback you submitted.


<CodeGroup>
```json Specify Project and People
{
  "data.class": "person",
  "project_name": "my_project"
}
```
```json Specific Feedback ID
{
  "feedback_id": "my_id"
}
```
</CodeGroup>

#### Operations

- [$exists](https://www.mongodb.com/docs/manual/reference/operator/query/exists/)
- [$gte](https://www.mongodb.com/docs/manual/reference/operator/query/gte/)
- [$gt](https://www.mongodb.com/docs/manual/reference/operator/query/gt/)
- [$lte](https://www.mongodb.com/docs/manual/reference/operator/query/lte/)
- [$lt](https://www.mongodb.com/docs/manual/reference/operator/query/lt/)
- [$ne](https://www.mongodb.com/docs/manual/reference/operator/query/ne/)
- [$in](https://www.mongodb.com/docs/manual/reference/operator/query/in/)

<RequestExample>
```bash Request
curl "https://v1-api.sievedata.com/v1/feedback/search" \
  -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data.class": "person",
    "project_name": "my_project",
    "type": "addition"
  }'
```
</RequestExample>
