---
title: "Corrections"
api: "POST https://api.sievedata.com/v1/feedback/correct_object"
---

Submit a correction of any given set of objects in a video as a way to correct mistakes made by the system. Feedback is specified in the `corrections` parameter in the same format as the payloads returned by the [/query](/query/metadata) endpoint. The system will automatically make corrections to the data specified and then improve itself automatically based on the feedback.

Each correction must specify the `object_id` of the object to be corrected. All other fields depend on the project schema.

### Parameters

<Param body="corrections" type="list" required>
  List of corrections to make
</Param>

<Param body="incorporate_new_classes" type="boolean" required>
  Whether or not the user intends to add new `classes` that don't yet exist in
  the project. If true, the system accepts any new classes specified but it
  defaults to false.
</Param>

<RequestExample>

```bash Terminal
curl 'https://api.sievedata.com/v1/feedback/correct_object' \
  -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "corrections": [
      {
        "object_id": "12345",
        "class": "person"
      }
    ]
  }'
```

```bash New bounding box position
curl 'https://api.sievedata.com/v1/feedback/correct_object' \
  -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "corrections": [
      {
        "object_id": "12345",
        "temporal": [
          {
            "bbox": {
              "position": {
                "x1": 5,
                "x2": 50,
                "y1": 5,
                "y2": 10
              }
            },
            "frame_number": 5
          }
        ]
      }
    ]
  }'
```

</RequestExample>

<ResponseExample>

```json Example
{
  "description": "All objects corrected successfully.",
  "successful_corrections": [12345],
  "skipped_corrections": []
}
```

</ResponseExample>
